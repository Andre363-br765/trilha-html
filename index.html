História dos computadores

voltar

1ª Geração (1940 - 1956)

  1. Harvard Mark I (1944) – criado para realizar cálculos complexos. Não foi o primeiro computador do mundo, mas é muito relevante.
  2. Colossus (1946) – usado para cálculos e criptografia durante a Segunda Guerra Mundial. Foi construído na Inglaterra e usado para decifrar códigos alemães durante a Segunda Guerra Mundial.
  3. ENIAC (1946) – criado nos EUA; muito mais automático. Aqui surgiu o termo bug, pois as válvulas esquentavam e atraíam insetos, o que acabava travando a máquina.

Esses computadores eram gigantescos e pesavam toneladas.

2ª Geração (1959 - 1965)

As válvulas foram substituídas por transistores, reduzindo muito o tamanho dos computadores, apesar de ainda serem enormes mas muito mais eficientes.

3ª Geração (1965 - 1970)

Surgem os circuitos integrados (chips com transistores, resistores e capacitores). Os computadores diminuem ainda mais.

4ª Geração (a partir de 1971)

Aparecem os microprocessadores e começam os computadores pessoais (PCs). Aqui a computação se torna acessível ao público.

5ª Geração

Alguns dizem que já estamos na 5ª geração, outros dizem que ainda não. Ela envolve inteligência artificial, internet, sistemas avançados e criptomoedas.

História da Internet

Antes da internet

A comunicação remota começou com o telégrafo e o código Morse. Em 1858 foram criados os primeiros cabos submarinos para comunicação entre continentes.

Década de 1950 – Guerra Fria

A corrida espacial acelerou o desenvolvimento tecnológico e pesquisas avançadas de comunicação.

Licklider

Ele propôs a ideia de uma “Rede Intergalática de Computadores”. Era uma teoria muito avançada para a época.

Surgimento da ARPANET (1969)

A primeira rede de computadores. A primeira palavra enviada foi “LO” (tentaram enviar “LOGIN”, mas caiu após as duas primeiras letras).

TCP/IP – 1973

Criado por Vinton Cerf e Robert Kahn, tornou a rede mais segura, organizada e confiável.

World Wide Web – 1989

Criada por Tim Berners-Lee, combinando hipertexto com a internet. Sem isso, a internet seria completamente desorganizada.
Hoje acessamos sites como: Google.

Grandes nomes envolvidos

  • Grace Murray Hopper
  • Joseph Carl Robnett Licklider
  • Robert E. Kahn
  • Tim Berners-Lee
  • Marc Andreessen
  • Bill Gates
  • Steve Jobs

O que são Clients?

voltar

Na web, chamamos de client tudo aquilo que consome informações de um servidor. Pode ser um computador, celular, tablet ou até uma TV inteligente.

Client x Servidor

A internet é formada por computadores interligados. Nela temos:

  • Client – quem faz o pedido (usuário/dispositivo).
  • Servidor – quem guarda e serve as informações.

Quando você acessa um site:

  1. O client faz uma requisição ao servidor.
  2. O servidor processa o pedido.
  3. Ele envia arquivos HTML, CSS, JS e imagens.
  4. O navegador interpreta tudo e exibe o site.

Cache

O navegador guarda alguns arquivos no dispositivo, chamado cache. Isso deixa o carregamento mais rápido e evita baixar tudo novamente.

Papel do Front-end

O desenvolvedor de front-end precisa pensar em como o site vai funcionar em cada dispositivo e navegador:

  • Celular, computador, tablet, TV.
  • Chrome, Firefox, Edge, Safari etc.
  • Responsividade – ajustar a tela conforme o tamanho.
  • Compatibilidade – algumas tags podem não funcionar em todos os navegadores.

O W3C define os padrões que garantem que os sites funcionem de forma parecida em todos os navegadores.

Há sites, como o W3Schools , que mostram o suporte de cada tag.

Navegadores

Navegadores são programas instalados no dispositivo que interpretam arquivos como HTML, CSS, JS, PDF e outros. Cada empresa segue os padrões do W3C, mas ainda assim existem pequenas diferenças entre eles.

Aplicação Web x Aplicativo Instalado

Quando você usa o Facebook ou o YouTube pelo navegador, isso é uma aplicação web, feita com HTML, CSS e JavaScript.

Já quando usa o aplicativo instalado no celular, isso é um app nativo, criado com linguagens como Kotlin ou Swift.

Eles fazem coisas parecidas, mas funcionam de formas diferentes.

Dispositivos Móveis

Existem muitos tipos de dispositivos:

  • Celulares
  • Tablets
  • Computadores
  • TVs modernas
  • Diversos tamanhos de tela

Por isso, um desenvolvedor precisa entender seu público e criar experiências que funcionem bem em todos os cenários.

O que são Servers?

voltar

Os servers (servidores) são computadores preparados para atender muitos usuários ao mesmo tempo. Eles funcionam de forma diferente dos clients, pois precisam lidar com muita carga, muitas requisições e precisam ser rápidos e estáveis.

Um servidor pode funcionar pela internet ou apenas em uma rede local, servindo outras máquinas próximas. Eles são usados para armazenar imagens, vídeos, arquivos, sistemas de segurança, bancos de dados e para hospedar serviços como Gmail, sites, streaming e muito mais.

Servidores em Data Centers

Os servidores mais poderosos ficam em data centers, locais com máquinas gigantes e robustas, preparados para milhões de acessos diários.

Hardware e Software de um Servidor

Parte física (hardware)

  • Memória RAM
  • Armazenamento: HD, SSD, NVMe
  • Processadores potentes
  • Fontes de energia redundantes
  • Conexões de rede rápidas

Parte lógica (software)

  • Sistemas operacionais: Linux, Debian, Fedora, Ubuntu Server, Windows Server
  • Servidores web: Apache, Nginx
  • Softwares de monitoramento
  • Gerenciadores de banco de dados

Como funciona uma requisição

Cliente → requisição HTTP → Servidor

Servidor → HTML, CSS, JS → Cliente

Entendendo isso, fica mais fácil contratar o serviço de hospedagem adequado.

Tipos de Servidores e Problemas Comuns

  • Servidor de arquivos
  • Servidor de segurança
  • Servidor de e-mail como Gmail
  • Servidor web
  • Servidor de banco de dados
  • Servidor proxy
  • Firewall
  • DNS

Servidor Proxy

O proxy funciona como um intermediário entre o usuário e a internet. Ele verifica regras de acesso e pode bloquear requisições na ida ou na volta.

Firewall

O firewall atua como uma barreira de segurança, bloqueando acessos externos maliciosos que tentam chegar ao servidor.

Internet x Intranet

A internet é formada por computadores do mundo todo conectados. Já a intranet é uma rede interna privada, acessível apenas dentro de um local, como uma empresa.

DNS

O DNS traduz domínios em endereços IP. Exemplo: google.com → IP numérico.

Domínios .br são brasileiros. Domínios .com surgiram nos Estados Unidos, onde a internet nasceu.

Hospedagem de Sites

Hospedar um site significa enviar seus arquivos para um servidor, para que os clientes possam acessá-lo. Existem diversas empresas especializadas nisso.

Domínios

O domínio é o nome do site. Ele precisa ser registrado e tem custo. Se outra pessoa registrar antes, você pode ter que comprar dela.

SSL

SSL é o certificado que garante uma conexão segura, identificado pelo cadeado do navegador: https://.

FTP

O FTP é um método antigo, mas ainda usado, para enviar e baixar arquivos diretamente no servidor, usando usuário, senha e um programa específico.

Servidores combinados

Às vezes, uma máquina faz várias funções ao mesmo tempo, mas isso pode causar lentidão. Por isso, empresas grandes usam servidores separados para cada função.

Linguagens de Programação

voltar

Linguagem de programação é uma linguagem específica e formal usada para criar instruções que geram programas (softwares). Cada linguagem possui regras próprias.

As 5 linguagens mais populares (segundo RedMonk)

  1. JavaScript
  2. Python
  3. Java
  4. PHP
  5. C#

Essas linguagens aparecem no topo das pesquisas por serem muito usadas em desenvolvimento web, mobile, nuvem e aplicações diversas.

Linguagens Server-Side

Linguagens server-side são executadas no servidor, e não no navegador do usuário. Elas processam dados, acessam bancos de dados e constroem páginas dinâmicas.

  • PHP
  • Python (Django, Flask)
  • Node.js (JavaScript no servidor)
  • Java (Spring)
  • C# (ASP.NET)
  • Ruby
  • Go

Cada linguagem funciona melhor em certos tipos de servidor. Algumas rodam em qualquer ambiente; outras exigem uma configuração específica.

Linguagens Client-Side

Linguagens client-side rodam diretamente no navegador do usuário. Hoje, a única linguagem realmente client-side é:

JavaScript

É o navegador que interpreta o JS e executa as ações na página.

JavaScript dos dois lados

JavaScript é especial porque pode rodar tanto no lado do cliente (browser), quanto no lado do servidor.

Isso acontece graças ao motor V8 e ao ambiente Node.js, que permitem que o JavaScript seja executado fora do navegador.

Resumo geral

  • Linguagem de programação: regras para criar software.
  • Server-side: roda no servidor → PHP, Python, Java, Node.js etc.
  • Client-side: roda no navegador → JavaScript.
  • JavaScript: funciona dos dois lados (client + server).

Como é uma estrutura básica de um HTML?

voltar

HTML não é uma linguagem de programação, mas uma linguagem de marcação. Ela serve para estruturar conteúdo na web, assim como arquivos CSV, XML ou Markdown.

O que é uma tag

  • Textos e elementos são delimitados por tags que o navegador entende.
  • Cada tag possui função ou comportamento específico.
  • Podem ser criadas em Bloco de Notas, VSCode ou na ferramenta de desenvolvedor (F12 no Chrome).

Exemplos de tags

  1. <i> – texto em itálico
  2. <strong> – texto em negrito
  3. <u> – texto sublinhado
  4. <mark> – texto marcado/destacado
  5. <sub> – texto subscrito
  6. <p> – parágrafo
  7. <blockquote> – citação de texto

Algumas tags não precisam de fechamento, como <input> e <img>.

Atributos das tags

  • Globais: funcionam na maioria das tags (ex.: id, class)
  • Específicos: só funcionam em algumas tags (ex.: src em <img>)

Hierarquia e tipografia

  • <h1> até <h6> – títulos e subtítulos
  • <p> – parágrafos
  • <i>, <strong>, <u>, <mark>, <sub> – formatações dentro de parágrafos
  • <blockquote> – citação de texto

Exemplo de uso dentro de um parágrafo:

<p>Isso é um <strong>texto em negrito</strong> dentro de um parágrafo.</p>

Estrutura mínima de um HTML

<!DOCTYPE html>
<html lang="pt-BR">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>>Meu HTML</title>
  </head>
  <body>
    <!-- Conteúdo vai aqui -->
  </body>
</html>
Explicação:
<!DOCTYPE html> → define o tipo do documento
<html lang="pt-BR"> → idioma da página
<head> → informações da página (metadados)
<title> → título que aparece na aba do navegador
<body> → todo o conteúdo visível da página

Entendendo a diferença entre listas ordenadas e não ordenadas

voltar

Listas são usadas para organizar conteúdos que seguem uma ordem ou não. Existem dois tipos principais:

  • Lista ordenada (<ol>) – cada item é numerado automaticamente.
  • Lista não ordenada (<ul>) – cada item é marcado com uma bolinha ou outro marcador, sem numeração.

Para que o navegador entenda os itens da lista, é necessário usar a tag <li> dentro de <ol> ou <ul>.

Exemplos

Lista ordenada (<ol>):

  1. Item 1
  2. Item 2
  3. Item 3

Lista não ordenada (<ul>):

  • Item A
  • Item B
  • Item C
As listas ajudam a organizar informações de forma clara e legível para o usuário.

Links de referências

voltar

Links (ou hiperlinks) permitem navegar para outro lugar, seja na mesma página ou em outra. São criados com a tag <a> (âncora).

Atributos importantes:

  • href – define o destino do link (URL ou âncora na página).
  • target – define onde o link será aberto:
    • _blank → abre em uma nova aba/janela.
    • _self → abre na mesma aba (padrão).
  • title – exibe uma dica (tooltip) ao passar o mouse sobre o link.

Exemplos de uso:

  • Acesse o Google
  • Voltar ao índice
Os links ajudam a criar navegação entre páginas e seções, melhorando a experiência do usuário.
